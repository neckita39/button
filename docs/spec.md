# «Живой офис» — спека хакатон-прототипа

Дата: 2026-07-11. Команда: NFC-метки (чел 1), интеграция Б24 (чел 2), живая карта + QR-тур (мы).

## Идея одним абзацем

Веб-приложение на большом экране: изометрическая 2.5D-карта крыла офиса, где переговорки живут своей жизнью. Зелёная — свободна, синяя — занята (приход подтверждён NFC), жёлтая с тикающим таймером — ждёт подтверждения и на глазах у жюри освобождается, если никто не пришёл. Клик по комнате — карточка: фото, «Сейчас / Дальше», таймлайн дня. Второй экран — QR-сканер: наводишь камеру на QR на двери, поверх видео всплывает живая карточка комнаты (псевдо-AR).

Ключевой продуктовый кейс («кейс 14:48»): пустая красная переговорка — её освободили на 12 минут раньше или она свободна ещё час? Карточка отвечает: «Свободна ещё 42 мин · освободили на 24 мин раньше», дальше «16:00 — Демо спринта», плюс саммари дня («занята почти весь день»).

## Экраны

1. **index.html — карта** (большой экран/проектор)
   - Изометрический план крыла (SVG, рисуется из конфига `ROOMS` — заменим на реальный план по наброску).
   - Статусы: `free` (свободна), `awaiting` (бронь ждёт NFC, таймер обратного отсчёта), `occupied` (подтверждена, внутри аватарки участников).
   - Лента событий («14:51 Марс освобождена: никто не пришёл») + счётчик «спасено сегодня N ч».
   - Клик по комнате → карточка: фото (`photos/<id>.jpg`, фолбэк-градиент), вместимость и оснащение, блок «Сейчас» (с бейджем «освободили раньше на N мин»), блок «Дальше», полоска-таймлайн 9:00–19:00, саммари дня, кнопка «Постучаться» (тост-заглушка).
   - Демо-панель по клавише `D`: скорость времени ×1/×30/×120, «отсканировать метку», «сброс».

2. **scan.html — QR-тур** (телефон/вебкамера ноутбука)
   - `getUserMedia` + jsQR (вендорен локально, `vendor/jsQR.js`, без CDN — на площадке может не быть интернета).
   - jsQR отдаёт углы QR в кадре → карточка комнаты рисуется прямо над кодом и следует за ним (псевдо-AR, без капризного AR-трекинга).
   - QR кодирует просто `room:<id>` (или URL `scan.html#<id>`).
   - ⚠️ Камера требует secure context: **демо через localhost с вебкамерой ноутбука** (печатаем QR на бумаге). Для телефона — туннель (cloudflared/ngrok), если позволит сеть площадки.

## Модель данных

```js
Room  = { id, name, cap, features[], rect:{x,y,w,h}, door:{x,y} }   // метры, сетка крыла
Event = { id, title, organizer, attendees[], start, end,            // минуты от полуночи
          checkedInAt|null, released:null|'noshow'|'early', releasedAt|null }
State = { status:'free'|'awaiting'|'occupied', current, next, deadline,
          freeUntil, releasedEarly:{minutes}|null, occupants[], schedule[],
          summary }  // «Занята почти весь день» и т.п.
```

Правила статусов (grace-период `GRACE_MIN = 6` минут от начала события):
- есть текущее событие и `checkedInAt` → `occupied`;
- есть текущее событие, нет чек-ина, `now < start+grace` → `awaiting`, дедлайн `start+grace`;
- дедлайн истёк → событие помечается `released:'noshow'`, комната → `free`, запись в ленту, тост «Б24: переговорка удалена из события», счётчик спасённых часов += остаток брони.

## Интерфейс данных (для стыковки с Б24)

Всё UI-ядро говорит только с движком `shared/core.js` (глобал `LivingOffice`, классические скрипты — ES-модули не работают с file://):

```js
const engine = LivingOffice.createEngine();
engine.rooms; engine.now(); engine.getState(roomId);
engine.getFeed(); engine.getSavedMinutes();
engine.on('tick'|'change'|'feed'|'toast', cb);
engine.setSpeed(x); engine.checkIn(roomId); engine.reset();
engine.setSchedule(roomId, events);   // ← точка стыковки с Б24
```

Мок-движок симулирует день. Для боевого режима коллега-интеграция кормит `setSchedule()` данными REST:

- `calendar.event.get` — события за период: параметры `type` (user/group/company_calendar), `ownerId`, `from`, `to`, `section[]`. Переговорка живёт в событии как ресурс/LOCATION — фильтруем по ней.
- `calendar.event.get.nearest` — ближайшие события (для «Дальше»).
- Если бронь через новый модуль Бронирования: `booking.v1.resource.list` + методы `booking.v1.booking.*`.
- Пуш об удалении комнаты из события (no-show) у коллеги уже есть — тот же момент шлём в наш фид (вебхук/поллинг).

## Демо-сценарий (мок, старт симуляции 14:46)

- **Марс**: бронь 14:45–15:45 «Синк по интеграции», никто не отсканировал NFC → жёлтая, таймер до 14:51. На ×30 освобождается через ~10 сек на глазах жюри: вспышка, лента, тост «Б24: комната удалена», +59 мин к счётчику. Кнопкой демо-панели можно наоборот «отсканировать метку» — комната синеет, появляются аватарки.
- **Венера**: 14:30–15:30 «1:1», чек-ин в 14:32 → синяя, аватарки внутри.
- **Юпитер**: «Демо спринта» отпустили в 14:36 вместо 15:00 → кейс 14:48: «Свободна ещё 1 ч 12 мин · освободили на 24 мин раньше».
- **Сатурн**: встречи почти весь день → саммари «Занята почти весь день».
- **Плутон**: свободен весь день.
- Счётчик спасённого предзаполнен утренними релизами (~2 ч 40 мин).

## Визуал

Три направления собираются параллельно, выбираем по скриншотам:
A «диспетчерская ночью» — тёмная, занятые комнаты светятся как окна дома;
B «архитектурный макет» — светлая, чертёжные аннотации, комнаты как макет на столе;
C «игровая» — Two Point Office, сочные цвета, комнаты с настроениями.

План крыла и фото комнат — заглушки до прихода реального наброска и фоток
(меняются только конфиг `ROOMS` в core.js и файлы в `photos/`).
