# Живой офис — карта переговорок Битрикс24 (хакатон)

Живая изометрическая 2.5D-карта этажа офиса + QR-тур по переговоркам.
Работает поверх мок-движка, стыкуется с Битрикс24 через REST (см. «Стыковка»).
Стек: ванильный ES5 JS + SVG, без сборки и без внешних ресурсов (CDN запрещён —
демо должно работать без интернета). Все страницы открываются с localhost.

## Запуск и проверка

```sh
./start.sh                      # python3 -m http.server 8123
# карта:    http://localhost:8123/
# сканер:   http://localhost:8123/scan.html   (камера — только localhost/https)
# QR-метки: http://localhost:8123/qr.html     (печать, Cmd+P)
```

Синтаксис-проверка инлайн-скриптов (обязательно после правок HTML):
```sh
awk '/APP-START/,/APP-END/' index.html | sed '1d;$d' > /tmp/ci.js && node --check /tmp/ci.js
node --check shared/core.js
```
Визуальная проверка — только в браузере (Playwright): скриншоты, НАСТОЯЩИЕ клики
(`browser_click`, не synthetic dispatchEvent — они врут про pointer events).
Фото проверять с кэш-бастером (`?v=...`) — браузер агрессивно кэширует.

## Структура

- `shared/core.js` — ядро: конфиг этажа (ROOMS/FUNZONES/WALLS/DESKS/DECOR/AREAS,
  координаты в клетках эскизов `docs/plan*.jpg`), изометрия (`LivingOffice.iso`),
  движок симуляции дня (статусы, grace-таймер NFC, no-show release, лента,
  счётчик спасённых минут). UI говорит ТОЛЬКО с движком.
- `index.html` — карта: рендер сцены в SVG, карточка комнаты, лента, тосты Б24,
  пан/зум, демо-панель (клавиша D). Mobile-first (карточка — bottom sheet).
- `scan.html` — QR-тур: getUserMedia + jsQR (вендорен), псевдо-AR карточка,
  фолбэк «Без камеры», deep-link `scan.html#<room-id>`.
- `qr.html` — печатный лист QR-меток (кодируют `<origin>/scan.html#<id>`).
- `shared/b24.js` — адаптер живых данных: `?api=<url>` на любой странице.
- `photos/<id>.jpg` — фото комнат (ASCII-имена = id комнат), `photos/inter/` —
  интерьеры для референса.

## Правила, выстраданные в бою

- Ориентация фото: браузер применяет EXIF сам — НЕ «чинить» повороты по
  сторонним рендерам, ground truth только браузер.
- Клик vs пан на карте: НИКАКОГО `setPointerCapture` на svg — он съедает клики
  по комнатам. Драг отличается порогом 6px (`consumeDrag()`), гард стоит на всех
  кликабельных объектах.
- Порядок отрисовки сцены — фазами: задние стены → зона Разработки → стена →
  Холл → стена → QA (`zonePhase`). Чистая сортировка по x+y режет кубы стенами.
- Цвет куба = цвет стекла комнаты в жизни (accent), НИКОГДА не меняется от
  статуса. Статус — свет: occupied светится изнутри, awaiting — янтарная аура
  с таймером `NFC m:ss`, free — тихий.
- ES-модули нельзя (file:// без CORS), только классические скрипты и ES5.
- Сообщения пользователю — по-русски, без канцелярита.

## Стыковка с Б24 (зона коллеги-интеграции)

`engine.setSchedule(roomId, events)`; события `{id,title,organizer,attendees[],
start,end,checkedInAt,released,releasedAt}`, время — минуты от полуночи.
`shared/b24.js` принимает и упрощённый формат `[{room,busy,by,title,until}]`.
REST-методы: `calendar.event.get` (type/ownerId/from/to/section),
`calendar.event.get.nearest`; новый модуль брони — `booking.v1.*`.
NFC-подтверждение — зона другого коллеги, мы только отображаем.
