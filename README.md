# Живой офис

Хакатон-прототип: живая изометрическая карта офиса + QR-тур по переговоркам.
Работает поверх мок-движка; стыкуется с Битрикс24 через REST (см. «Стыковка с Б24»).

## Запуск

```sh
./start.sh          # или: python3 -m http.server 8123
```

| Страница | URL | Что это |
|---|---|---|
| Карта | http://localhost:8123/ | Большой экран: живая карта крыла |
| Варианты | http://localhost:8123/prototypes/variant-a.html (b, c) | Три визуальных направления |
| QR-тур | http://localhost:8123/scan.html | Камера + псевдо-AR карточка комнаты |
| QR-метки | http://localhost:8123/qr.html | Печать меток на двери (Cmd+P) |

Камера в QR-туре работает только на `localhost` или `https`. Для демо проще
всего: ноутбук с вебкамерой + распечатанные метки. Для телефона нужен туннель
(`cloudflared tunnel --url http://localhost:8123` или ngrok).

## Демо-сценарий для жюри (3 минуты)

1. **Карта на проекторе.** «Это наш этаж прямо сейчас». Кубы — реальные
   стеклянные переговорки офиса, цвет куба = цвет стекла в жизни.
2. **Кейс 14:48.** Кликнуть «Зелёную»: свободна ещё 1 ч 12 мин, бейдж
   «освободили на 24 мин раньше», дальше «16:00 — Груминг». Ни один календарь
   так не отвечает на вопрос «а можно её занять?».
3. **Гвоздь: no-show.** «Красная» пульсирует янтарным — бронь ждёт NFC-метку.
   Нажать `D` → скорость ×30. Таймер дотикивает до 14:51 — вспышка, комната
   свободна, в ленте «никто не пришёл», пуш «Битрикс24: переговорка удалена из
   события», счётчик «спасено сегодня» прыгает. Это работа всей связки:
   NFC-метка (наша), интеграция Б24 (наша), карта (наша).
4. **Спасение.** Сброс сценария (`D` → «Сброс»), снова ×30, но теперь нажать
   «Отсканировать метку в Красной» — куб загорается тёплым светом, внутри
   появляются аватарки. Так выглядит подтверждение прихода.
5. **QR-тур.** Открыть scan.html, навести вебкамеру на распечатанную метку —
   карточка комнаты всплывает прямо над QR и следует за ним. «Стоишь у двери —
   видишь, свободна ли она на самом деле».
6. Финал: «Две метки на каждой двери: NFC отвечает "я пришёл",
   QR отвечает "что тут происходит"».

## Структура

```
shared/core.js    — конфиг крыла (комнаты, фан-зоны), изометрия, движок симуляции
prototypes/       — variant-a/b/c.html: три визуальных направления карты
scan.html         — QR-тур (jsQR, псевдо-AR, фолбэк «Без камеры»)
qr.html           — печатный лист QR-меток
photos/           — фото комнат (ASCII-имена = id комнат)
docs/spec.md      — продуктовая спека
docs/plan.jpg     — набросок плана крыла
vendor/           — jsQR, qrcode-generator (локально, без CDN)
```

## Стыковка с Б24 (для интеграции)

UI говорит только с движком. Точка стыковки — `engine.setSchedule(roomId, events)`,
события в формате `{id, title, organizer, attendees[], start, end, checkedInAt,
released, releasedAt}` (время — минуты от полуночи).

REST-методы: `calendar.event.get` (type/ownerId/from/to/section — события за
период, переговорка = ресурс/LOCATION события), `calendar.event.get.nearest`
(ближайшие), при новом модуле бронирования — `booking.v1.resource.list` +
`booking.v1.booking.*`. Пуш об удалении комнаты из события (no-show) шлём в тот
же канал — карта его показывает лентой и тостом.

## Как поменять план крыла / фото

- Геометрия: `shared/core.js`, массивы `ROOMS` и `FUNZONES` — координаты в
  клетках эскиза (`docs/plan.jpg`), `rect` комнаты и `door` на её грани.
- Фото: `photos/<id>.jpg` (карточки берут их по id комнаты).
